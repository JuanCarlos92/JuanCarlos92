### 6º. Creación de las Rutas

Para que la aplicación responda a las solicitudes HTTP, es necesario definir las reglas de navegación en el archivo **`routes/web.php`**. Aquí especificarás qué controlador se encargará de gestionar cada URL.

**Ejemplo de configuración:**

```php
use App\Http\Controllers\PostController;
use Illuminate\Support\Facades\Route;

// Ruta para el listado principal
Route::get('/', [PostController::class, 'index'])->name('posts.index');

// Ruta para ver un post específico mediante su ID
Route::get('/posts/{id}', [PostController::class, 'show'])->name('posts.show');

```

**¿Qué significan estas rutas?**

* Al acceder a la **raíz (`/`)**, Laravel invocará el método `index` del controlador `PostController`.
* Al solicitar un **post específico (`/posts/1`)**, se utilizará el método `show`, pasando el ID como parámetro.
* El uso de `->name()` permite asignar un nombre a la ruta para referenciarla fácilmente en toda la aplicación.

---

> **Consejo de escalabilidad:**
> A medida que la aplicación crezca, es recomendable agrupar rutas relacionadas y aplicar **middleware** para gestionar de forma centralizada aspectos como la autenticación o la seguridad.
