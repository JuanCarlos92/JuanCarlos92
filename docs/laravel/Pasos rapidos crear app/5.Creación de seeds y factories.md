
### 5º. Creación de Seeds y Factories

Las **factories** y **seeds** son herramientas fundamentales para poblar tu base de datos con datos de prueba, lo que es muy útil durante el desarrollo.

* **Factory:** Define cómo se generarán datos falsos para un modelo.
* **Seeder:** Permite ejecutar la factory y poblar la base de datos de forma masiva.

#### Ejemplo de Factory
Edita el archivo ubicado en `database/factories/PostFactory.php` para especificar valores aleatorios utilizando la librería **Faker**:

```php
public function definition() {
    return [
        'titulo' => $this->faker->sentence(),         // Oración aleatoria
        'contenido' => $this->faker->paragraph(),     // Párrafo aleatorio
        'imagen' => $this->faker->imageUrl(640, 480), // URL de imagen (640x480)
    ];
}

```

#### Ejemplo de Seeder

Crea o edita el archivo en `database/seeders/PostSeeder.php` para indicar cuántos registros quieres generar:

```php
use App\Models\Post;

public function run() {
    // Genera 20 posts de ejemplo utilizando el factory definido arriba
    Post::factory(20)->create();
}

```

---

#### ¿Cómo utilizar este seeder?

**1. Registro en DatabaseSeeder:**
Para que se ejecute al correr el comando de seeding general, debes llamar al `PostSeeder` desde el archivo `database/seeders/DatabaseSeeder.php` dentro del método `run()`:

```php
public function run() {
    $this->call(PostSeeder::class);
}

```

**2. Ejecución del Seeder:**
Finalmente, ejecuta el siguiente comando en tu terminal para refrescar la base de datos y poblarla:

```bash
php artisan migrate:fresh --seed

```

> **Nota:** El comando `migrate:fresh` elimina todas las tablas y las vuelve a crear desde cero, mientras que la opción `--seed` se encarga de rellenarlas automáticamente.
